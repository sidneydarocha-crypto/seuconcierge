Fluxo de conversa

Como o agente processa cada mensagem — do recebimento no WhatsApp até a resposta final ao hóspede.

💬
Mensagem recebida Webhook
WhatsApp envia POST para nosso endpoint. Gateway valida a assinatura HMAC e extrai o payload.
🔍
Identificação do hotel Multi-tenant
Identifica qual hotel/pousada é dono do número. Carrega o perfil da propriedade (nome, serviços, políticas, tom de voz).
💾
Contexto do hóspede Memory
Busca o histórico de conversa do número no Redis. Se novo hóspede, cria perfil. Garante continuidade entre sessões.
🧠
Processamento pela IA Claude API
Monta o system prompt com dados do hotel + histórico. Envia para Claude claude-sonnet-4-20250514. O modelo decide se precisa chamar uma tool (verificar disponibilidade, criar reserva, etc).
🔧
Tool Use (se necessário) PMS API
Se o modelo detectou intenção de reserva ou consulta de disponibilidade, chama a API do PMS (Cloudbeds/Omnibees) para obter dados reais em tempo real.
🤔
Decisão: IA ou humano? Escalada
O modelo avalia se consegue resolver ou se deve escalonar. Critérios: reclamação grave, pedido explícito, dúvida complexa de contrato.
📤
Envio da resposta WhatsApp API
Formata a resposta para o WhatsApp (texto, botões, listas). Envia via Meta Cloud API. Salva a troca no histórico.
📊
Analytics & triggers Background
Registra métricas (tempo de resposta, intent detectado). Dispara automações: check-in D-1, solicitação de review pós-checkout.
